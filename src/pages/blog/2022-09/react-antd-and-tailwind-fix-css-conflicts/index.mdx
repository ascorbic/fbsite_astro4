If you're using React, [Tailwind](https://tailwindcss.com/) and [Ant Design](https://ant.design) (and probably most of the others UIKIT available in the market) you may encounter some CSS conflicts.

For example, the AntD `Modal` component shows the "OK" and "Cancel" buttons by default in its footer:


<Code>
```ts
 <Modal title="Add city" onOk={} onCancel={} />
```
</Code>

As you can see in the screenshot below, the OK button is not displayed as it should (I mean it should be _blue_):

<DesktopWindow center>
![AntD and Tailwind conflict](https://res.cloudinary.com/my-notes-demo/image/upload/v1656813412/tutorial-content/dev.to/2022-07-antd-tailwind-conflicts/2022-07-03_03-55-26.png)
</DesktopWindow>

In fact Tailwind applies a transparent background color, while AntD should apply a blue background:

<DesktopWindow center>
![](https://res.cloudinary.com/my-notes-demo/image/upload/v1656813786/tutorial-content/dev.to/2022-07-antd-tailwind-conflicts/2022-07-03_04-02-21.png)
</DesktopWindow>

To solve the issue we can disable Tailwind [Preflight](https://tailwindcss.com/docs/preflight), a set of base styles that are designed to smooth over cross-browser inconsistencies.

<Code>
```ts
// tailwind.config.js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./src/**/*.{js,jsx,ts,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [
    // ...
  ],
  corePlugins: {
    preflight: false // <== disable this!
  },
}
```
</Code>

And the problem is magically solved:
